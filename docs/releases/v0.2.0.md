# v0.2.0 Release Notes

## 概览
v0.2.0 主要增强“可用性”和“稳定触发工具调用”的概率：不用改文件也能覆盖配置，并把工具协议明确地告诉模型。

## 这次增加了什么（What）
- CLI 参数覆盖配置：`--base-url` / `--model` / `--api-key` / `--timeout-seconds` / `--memory-file` / `--max-turns` / `--config` / `--print-config`。
- 环境变量覆盖：`AGENT_BASE_URL` / `AGENT_MODEL` / `AGENT_API_KEY` / `AGENT_TIMEOUT_SECONDS` / `AGENT_MEMORY_FILE` / `AGENT_MAX_TURNS`。
- 固定注入 `system` prompt：明确“需要工具时只输出 JSON”并列出可用工具与参数 schema。
- CLI 命令 `:reset`：清空 `memory.json`（调试/演示更方便）。

## 为什么要增加（Why）
- 仅靠 README 描述工具协议，模型未必每次都遵循；缺少“系统提示 + 工具清单”会导致工具触发不稳定。
- 每次切换环境（不同 baseUrl/model/apiKey）都要改 `agent.json` 不方便，也容易把配置提交错。

## 解决了什么问题（Problem Solved）
- 提升工具调用可控性：把“如何调用工具”的规则变成强约束的 system 指令。
- 提升可移植性：同一个 jar 在不同机器/网关下可通过 CLI/env 快速运行，不必改配置文件。
- 提升调试效率：`:reset` 可快速清空历史，避免旧上下文影响演示结果。

## 设计与实现要点（How）
- `CliArgs` 做最小化参数解析（不引入外部 CLI 框架）。
- `Agent.respond()` 在每次调用 LLM 前构造 `ctx = [systemPrompt] + history`，确保工具说明始终在上下文中。
- `ToolRegistry.toolsPrompt()` 输出工具名/描述/参数 schema 作为系统提示的一部分。

## 使用方式（Usage）
```bash
mvn -q -DskipTests package
java -jar target/java-agent-0.2.0.jar --base-url http://127.0.0.1:8317/v1 --model gpt-5.2
```
