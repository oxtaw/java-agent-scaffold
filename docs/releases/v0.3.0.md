# v0.3.0 Release Notes

## 概览
v0.3.0 新增两个“本地安全文件系统工具”，用于在不复制粘贴的情况下让 Agent 查看项目文件/目录（仅允许相对路径，防止越权读文件）。

## 这次增加了什么（What）
- `fs.ls`：列出本地目录条目（相对路径）。
- `fs.read`：读取本地文本文件并截断输出（相对路径）。

## 为什么要增加（Why）
- 让 Agent 能“就地查看”工作目录下的文件内容/结构，便于调试配置、查看日志片段、阅读小型文档或代码。
- 默认不开放绝对路径和路径穿越，减少误读敏感文件的风险。

## 解决了什么问题（Problem Solved）
- 调试/演示时不再需要人工复制粘贴文件内容到对话里。
- 用最小权限的方式满足“读项目文件”需求，避免 `../../` 等路径逃逸。

## 设计与实现要点（How）
- 两个工具都只接受相对路径，并做 `normalize + startsWith(root)` 校验，阻止越出工作目录。
- `fs.read` 限制最大输出字符数，避免一次读取过大文件导致上下文污染。

## 使用方式（Usage）
模型在需要时会输出（示例）：
```json
{"tool":"fs.ls","args":{"path":"src/main/java","maxEntries":200}}
```
